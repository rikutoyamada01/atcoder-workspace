# コントリビューションガイドライン

atcoder-workspace プロジェクトへ関心を持っていただきありがとうございます！このプロジェクトは、AtCoderでの競プロ体験を向上させるためのオープンソースソフトウェアです。バグ報告、機能提案、コードの寄稿など、あらゆる形での貢献を歓迎します。

---

## 開発環境のセットアップ

本拡張機能の開発をローカルで行うには、以下の手順に従ってください。

### 必要な環境
- Node.js (推奨: v18 以上 / v20 LTS)
- Google Chrome ブラウザ (または Chromium ベースのブラウザ)

### セットアップ手順
1. **リポジトリをフォーク & クローンする**
   ```bash
   git clone https://github.com/YOUR-USERNAME/atcoder-workspace.git
   cd atcoder-workspace
   ```
2. **依存関係のインストール**
   ```bash
   npm install
   ```
3. **Monaco Editorのセットアップ**
   エディタの静的アセットをコピーするセットアップスクリプトを実行します。
   ```bash
   npm run setup
   ```
4. **Chromeへパッケージ化されていない拡張機能として読み込む**
   - Chromeで `chrome://extensions/` を開きます。
   - 右上の「デベロッパー モード」をオンにします。
   - 「パッケージ化されていない拡張機能を読み込む」をクリックし、プロジェクトのルートディレクトリを選択します。

## コードスタイルとルール

本プロジェクトでは、コードの一貫性を保つため、ESLint と Prettier を導入しています。

* **コードフォーマットのチェック**:
  ```bash
  npm run format:check
  ```
* **コードフォーマットの自動修正**:
  ```bash
  npm run format
  ```
* **Linterの実行**:
  ```bash
  npm run lint
  ```

コミットやプルリクエストを作成する前に、必ずフォーマットとLinterのチェックを通すようにしてください。

---

## テストの実行

Jest を使用したユニットテストが用意されています。変更を加えた際は、既存のテストが壊れていないか確認してください。

* **テストの実行**:
  ```bash
  npm test
  ```

新しい機能を追加する場合は、可能な限りテスト（`tests/` ディレクトリ内）も合わせて追加してください。

---

## プルリクエスト（PR）の提出手順

1. `master`（または `main`）ブランチから新しいフィーチャーブランチを作成します。
   ```bash
   git checkout -b feature/my-new-feature
   ```
2. 変更を加え、ローカルテスト・Linterがすべてパスすることを確認します。
3. コミットを作成します。コミットメッセージは簡潔かつ具体的で、変更内容が分かりやすいものにしてください。
4. ブランチを GitHub にプッシュし、プルリクエストを作成します。
5. バージョンアップを伴う機能追加の場合は、`manifest.json` のバージョン番号を更新（バンプ）してください。PRビルド時にCIによるバージョン重複チェックが実行されます。
