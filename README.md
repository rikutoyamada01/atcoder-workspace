AtCoderの問題ページを、エディタ・ローカルテスト・提出が統合された「ブラウザ上の開発環境（オンラインIDE）」に変えるGoogle Chrome拡張機能です。環境構築不要、サンプルのコピペ不要、画面・タブの移動不要で、問題画面を開いたその瞬間からコーディングとテストを快適に完結させることができます。

## 特徴と主な機能

### 1. 問題文一体型の分割ビューエディタ
- **シームレスなコーディング:** 左側に問題文、右側に高品質なコードエディタ（Monaco Editor）を配置。境界線はドラッグで自由にリサイズ可能です。
- **ダークテーマ対応:** OSやブラウザの設定に基づき、AtCoderのライト・ダークテーマを自動で検出し適用します。
- **快適な入力支援:** 括弧の自動補完や自動インデント、コード整形（フォーマット）などの入力補助を備えています。

### 2. ワンクリック自動ローカルテスト
- **サンプルの自動抽出:** 問題文内の「入力例」と「期待される出力値」を自動でスクレイピングします。
- **バックグラウンド実行:** AtCoder公式のカスタムテストAPIを利用して裏でコードを実行。各テストケースの実行状態や判定結果（AC / WA / RE / TLE）を一括で表示します。
- **インタラクティブな詳細確認:** 実行結果のヘッダーをクリックして開閉できるアコーディオンUIを採用。WA時の期待値と出力のテキスト差分表示、RE時の標準エラー出力（stderr）表示に対応しています。
- **スライダーによるコンソール開閉:** エディタ最下部の中央にあるスライダーボタン、または `Ctrl + J` ショートカットキーで、テスト結果を表示するコンソールパネルをいつでも瞬時に開閉できます。

### 3. データ消失を防ぐ自動保存
- キー入力の停止（1.5秒）またはタブ切り替え・ウィンドウ非表示のタイミングで、コードを `chrome.storage.local` に自動保存します。クラッシュや誤操作によるコード損失を防ぎます。

### 4. クイックナビゲーション
- エディタ上部の「前の問題」「次の問題」ボタンから、コンテストページに戻ることなく隣接する問題へ直接遷移できます。

## プロジェクト構成

```
atcoder-workspace/
 ├── manifest.json              # 拡張機能のマニフェスト定義
 ├── package.json               # 依存関係定義
 ├── scripts/
 │    ├── setup-monaco.js       # Monaco Editorのセットアップスクリプト
 │    └── toggle-feature.js     # ロードマップ機能のステータス管理
 ├── src/
 │    ├── content/
 │    │    ├── content.js       # 画面レイアウト構築、サンプル抽出、API実行ポーリング
 │    │    └── content.css      # スプリットレイアウトとトグル用スタイル
 │    └── editor/
 │         ├── editor.html      # エディタ画面UI定義
 │         ├── editor.js        # Monaco初期化、イベントリスナー、テスト結果の描画
 │         └── editor.css       # エディタのツールバー、テスト結果行などのスタイル
 └── docs/                      # 各種仕様書・ビジョンドキュメント
```

## 開発者向けセットアップ

ローカル環境で拡張機能をビルド・インストールし、開発を行うための手順です。

### 1. 依存関係のインストールとセットアップ
リポジトリをクローンした後、パッケージをインストールして Monaco Editor の静的アセットをコピーするセットアップスクリプトを実行します。

```bash
git clone https://github.com/rikutoyamada01/atcoder-workspace.git
cd atcoder-workspace
npm install
npm run setup
```

### 2. Chromeブラウザへのインストール
1. Google Chromeで `chrome://extensions/` （拡張機能管理画面）を開きます。
2. 右上の「デベロッパー モード」をオンにします。
3. 左上の「パッケージ化されていない拡張機能を読み込む」をクリックします。
4. 本プロジェクトのルートディレクトリ（`manifest.json` が配置されているフォルダ）を選択して読み込みます。

## 将来的なロードマップ

### フェーズ 2: エラー分析と学習支援
- **ジャッジ結果のローカル記録**: AC、WA、TLE、RE、CEなどの提出ステータスと履歴をローカルに保存。
- **学習メモ機能**: Markdown対応のメモパネルをエディタと切り替えて表示。

### フェーズ 3: ナレッジベースの構築
- **ライブラリ高速検索**: 過去のACコードやメモをアルゴリズムタグ等で整理し、再利用可能にするライブラリ機能。
- **復習管理機能**: 過去の間違いが多い問題を最適なタイミングで提示する再挑戦キュー。
- **データエクスポート**: 各問題のコードやメモを、Obsidian等のMarkdownベースのノートツールと連携可能な形式でエクスポートする機能。
